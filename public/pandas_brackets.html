Single column — returns a Series

💡
You can also write df.experiment using dot notation — but only if the column name has no spaces or special characters, and does not clash with a pandas method name. The bracket form always works.

Multiple columns — returns a DataFrame

Pass a list of strings. You get back a DataFrame — note the double brackets: the outer pair belongs to pandas, the inner pair creates the list.

python 
df[["experiment", "energy_tev"]]

# inner [ ] = Python list

⚠️
A very common mistake: df["experiment", "energy_tev"] — this passes a tuple, not a list, and raises a KeyError. You need the inner list: df[["experiment", "energy_tev"]].
03

Boolean filter — returns matching rows

Pass a boolean Series — a column of True/False values. Pandas returns only the rows where the value is True. The Series is almost always built from a comparison on the DataFrame itself.

python 
# step 1: build the boolean Series
mask = df["num_files"] > 400

# step 2: use it to filter
df[mask]

# or in one line:
df[df["num_files"] > 400]

⚡ interactive — build a filter

Choose a column, operator, and value. See the matching rows.

combining conditions 
# AND — both must be true — use &
df[(df["num_files"] > 400) & (df["energy_tev"] == 13.6)]

# OR — either must be true — use |
df[(df["energy_tev"] == 13.6) | (df["num_files"] < 100)]

# NOT — invert a condition — use ~
df[~(df["experiment"] == "ALICE")]
⚠️
Use & and |, not and and or. Python's and/or operate on single values; &/| operate element-wise across a whole Series. Also wrap each condition in parentheses — operator precedence will bite you without them.
04

Row slices — works like a list

Pass a Python slice to get rows by position. This is the one case where pandas behaves like a plain Python list. You almost always want .iloc[] or .loc[] for serious row access, but the slice shortcut is useful for quick inspection.

python 
df[0:3]      # rows 0, 1, 2 — stop index is exclusive
df[:5]       # first 5 rows — same as df.head(5)
df[-3:]      # last 3 rows
df[::2]      # every other row (step=2)
✅
For anything beyond quick inspection, prefer df.iloc[0:3] (position-based) or df.loc[0:2] (label-based). They are explicit and behave predictably when the index is not a clean 0, 1, 2 sequence.
05

Assigning new columns

Use bracket assignment to create or overwrite a column. The right-hand side can be a scalar, a list, or — most usefully — an expression over existing columns.

python 
# scalar — same value in every row
df["accelerator"] = "LHC"

# expression over an existing column
df["energy_gev"] = df["energy_tev"] * 1000

# boolean flag
df["high_volume"] = df["num_files"] > 500

# overwrite an existing column
df["num_files"] = df["num_files"].fillna(0)
💡
Assignment modifies the DataFrame in place — there is no return value. If you want to leave the original untouched, work on a copy: df2 = df.copy() before assigning.
06

Combining techniques

The real power comes from chaining these together. Filter rows, then select columns. Select columns, then filter. Assign a new column, then filter on it.

filter rows → select columns 
# get experiment names for high-volume runs only
df[df["num_files"] > 400][["experiment", "num_files"]]
new column → filter on it 
# add a flag, then filter on it
df["high_volume"] = df["num_files"] > 500
df[df["high_volume"]]
multiple conditions + column selection 
df[
  (df["energy_tev"] == 13.6) &
  (df["num_files"] > 100)
][["experiment", "run_year", "num_files"]]
